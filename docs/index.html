Installation

Step 1: Add JitPack Repository

In your settings.gradle.kts (root level), add the JitPack repository:

dependencyResolutionManagement {
    repositoriesMode.set(RepositoriesMode.FAIL_ON_PROJECT_REPOS)
    repositories {
        google()
        mavenCentral()
        maven { url = uri("https://jitpack.io") }
    }
}

Step 2: Add Dependency

In your app-level build.gradle.kts:

dependencies {
    implementation("com.github.NereyaHillel:InAppNotificationLibrary:1.1.0")
}

Step 3: Sync Gradle

Click "Sync Now" in Android Studio.

Quick Start

1. Initialize the SDK

Initialize in your Application class:

import com.example.inappnotifications.InAppNotifier

class MyApplication : Application() {
    override fun onCreate() {
        super.onCreate()
        InAppNotifier.init(this, "user_123")
    }
}
Important

Must be called before using other SDK functionality.

2. Register Device

lifecycleScope.launch {
    val success = InAppNotifier.registerDevice()
    if (success) Log.d("Notifications", "Device registered")
}

3. Add Permissions

The following permissions are automatically declared in the library manifest:

<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.READ_PHONE_STATE" />
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />

4. Fetch and Display Notifications

import com.example.inappnotifications.NotificationPosition

lifecycleScope.launch {
    val notifications = InAppNotifier.getNotifications()

    notifications?.forEach { notification ->
        InAppNotifier.showNotificationPopup(
            context = this@MainActivity,
            notification = notification,
            position = NotificationPosition.CENTER,
            positiveButtonText = "Accept",
            onPositiveClick = {
                Log.d("Notifications", "User accepted")
            }
        )
    }
}

API Reference

InAppNotifier Singleton

The main object for managing in-app notifications.

init(context: Context, userId: String)

Initializes the SDK with user identification and device information.

context
Android application context
userId
Unique user identifier

registerDevice(): Boolean

Registers the current device with the notification server.

Returns:
Boolean – true if successful

getNotifications(): List<InAppNotification>?

Fetches the list of pending notifications for the current user.

Returns:
List of InAppNotification objects or null on error

trackInteraction(notificationId: String, action: String): Boolean

Records user interaction with a notification.

notificationId
ID of the notification
action
Action type: "clicked", "dismissed", etc.

showNotificationPopup(...): Unit

Displays an in-app notification dialog with customizable buttons and positioning.

  • context – Android application context
  • notification – The InAppNotification to display
  • position – Position on screen: TOP, CENTER, BOTTOM (default: CENTER)
  • imageUrl – Optional image URL to display
  • fallbackImageRes – Drawable resource ID for failure
  • positiveButtonText – Optional positive button text
  • onPositiveClick – Optional callback for positive button
  • negativeButtonText – Optional negative button text
  • onNegativeClick – Optional callback for negative button
  • neutralButtonText – Optional neutral button text
  • onNeutralClick – Optional callback for neutral button
  • onDismiss – Optional callback when dialog is dismissed

reportCrash(crashDetails: String): Boolean

Reports a crash to the notification server. Called automatically for unhandled exceptions.

crashDetails
Stack trace and crash details

Data Models

InAppNotification

Represents a notification from the server.

data class InAppNotification(
    val _id: String,              // Notification unique ID
    val campaign_id: String,      // Campaign identifier
    val message: String,          // Message content
    val status: String,           // Status: "unread", "read", etc.
    val title: String             // Notification title
)

RegisterDeviceRequest

Device registration request sent to the server.

data class RegisterDeviceRequest(
    val device_model: String,     // Device model (e.g., "Pixel 6")
    val device_id: String,        // Unique device ID
    val user_id: String           // Associated user ID
)

CrashReportRequest

Crash report sent to the server when exceptions occur.

data class CrashReportRequest(
    val user_id: String,          // User experiencing crash
    val crash_details: String     // Stack trace details
)

Advanced Usage

Custom Dialog Positioning

Display notifications at different screen positions:

// Top position
InAppNotifier.showNotificationPopup(
    context = context,
    notification = notification,
    position = NotificationPosition.TOP
)

// Center (default)
InAppNotifier.showNotificationPopup(
    context = context,
    notification = notification,
    position = NotificationPosition.CENTER
)

// Bottom position
InAppNotifier.showNotificationPopup(
    context = context,
    notification = notification,
    position = NotificationPosition.BOTTOM
)

Complete Example

lifecycleScope.launch {
    val notifications = InAppNotifier.getNotifications() ?: return@launch

    notifications.forEach { notification ->
        InAppNotifier.showNotificationPopup(
            context = this@MainActivity,
            notification = notification,
            position = NotificationPosition.CENTER,
            positiveButtonText = "Learn More",
            onPositiveClick = {
                startActivity(Intent(this@MainActivity, DetailsActivity::class.java))
            },
            negativeButtonText = "Dismiss",
            onNegativeClick = {
                Log.d("Notifications", "Dismissed")
            },
            onDismiss = {
                Log.d("Notifications", "Dialog closed")
            }
        )
    }
}

Crash Reporting

The library automatically sets up a global crash handler. When an unhandled exception occurs, it will:

  1. Capture the crash details and stack trace
  2. Send the crash report to the server
  3. Delegate to the default exception handler
No Additional Setup Required

Crash reporting works automatically after SDK initialization.

Manual Crash Reporting

You can also manually report crashes:

lifecycleScope.launch {
    InAppNotifier.reportCrash("Custom crash details")
}

Error Handling

All API calls include built-in error handling with logging:

lifecycleScope.launch {
    try {
        val success = InAppNotifier.registerDevice()
        if (!success) {
            Log.d("Notifications", "Registration failed")
        }
    } catch (e: Exception) {
        Log.e("Notifications", "Exception during registration", e)
    }
}

Debugging with Logcat

Check Android Studio's Logcat with filter to view detailed diagnostic information:

adb logcat | grep "InAppNotifier"

Log Levels

  • DEBUG – General information and successful operations
  • WARN – Non-critical issues (e.g., not initialized)
  • ERROR – Critical errors with exceptions

Troubleshooting

"SDK not initialized" warning

Call InAppNotifier.init() before using APIs in your Application class onCreate.

Images not loading

Check that the URL is valid and provide a fallback drawable resource ID. The library uses Glide with intelligent fallback handling.

Dialog not showing

Ensure notification status is not "read". Only unread notifications are displayed.

Crashes not reported

Crashes report automatically after initialization. Check Logcat for details with tag filter: InAppNotifier

For additional debugging:

adb logcat | grep "InAppNotifier"
